Basics first Last updated: 2023-03-18

Taskzilla is designed to run on both Shared hosting (eg GoDaddy, InmotionHosting, etc) and Virtual Private Server (VPS) (eg DigitalOcean) or Dedicated Server, and is designed to run on Linux Operating Systems.

Server Requirements

This app is developed with Laravel 10 (Latest and Greatest). You need to make sure your server meets the following requirements for the basic setup. (We also check your server for these requirements in the installation wizard)

  • PHP >= 8.1
  • Ctype PHP Extension
  • cURL PHP Extension
  • DOM PHP Extension
  • Fileinfo PHP Extension
  • Filter PHP Extension
  • Hash PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PCRE PHP Extension
  • PDO PHP Extension
  • Session PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension
  • Apache mod_rewrite
  • memory_limit >= 100MB in PHP.ini
  • post_max_size >= 10MB in PHP.ini
  • upload_max_filesize >= 10MB in PHP.ini
  • file_uploads enabled in PHP.ini
  • MySQL Database

2: Installation Wizard

Taskzilla comes with an installation wizard which, makes it easy to install. To get started, do the following:

2.1: Start

  1. Download the package from your Envato Account
  2. Unzip the package to any convenient location on your local computer
  3. Inside the unzipped package, there is a zip file called upload_version_xx_xx.zip Upload the contents of this ZIP file into your website directory. You can use FileZilla to upload the ZIP file and unzip it in your server, or use any other upload means suitable to you. Just make sure you upload all the contents.
  4. After uploading, rename the .env.example file located in the root of the application to .env
  5. Now, go to your web browser and access your website url. This will automatically take you to the installation wizard as seen below:
  6. Click on "LET'S START" button to start the installation process

2.2: Server Requirements

Taskzilla is developed with Laravel v10 (The latest version and greatest). Therefore to get the script working, your server must meet these basic requirements for running a laravel application.

Besides the basic server requirements for running Laravel, there are other requirements that your server must meet, such as values from PHP.ini file to handle file uploads, folder permissions etc. On the "System Requirements" screen, you can see whether or not your server meets these requirements. Please ensure that you see a "Green Check mark" against each requirement before you proceed.

2.3: Database Connection

The script requires MySQL database. On your server, create a new MySQL database, copy the hostname, username, password and port of your database connection, and enter them in the "Database Settings" form as shown below:

Then click on "Proceed" to establish the connection. If the connection is successful, you will proceed to the next step. Otherwise, review the information entered and try again.

2.4: Mail Settings

You must provide working MAIL settings to enable the script send system emails. If your mail settings are wrong, users will not be able to receive invitations by email or change their passwords.

In the installer, configure your SMTP credentials. - You can get a free SMTP account from Mailgun NB: The free version of Mailgun should be sufficient for this script as it provides 10,000 free emails / month.

You can use a service like SMTPer.net to test your credentials before you proceed.

2.5: Create Admin User

In this stage, you can create your Administrator user account. This is the Super user and will be able to invite other users to join the site once logged in.

2.6: Finish and Login

Once your admin user is created, you will be redirected to the login screen. Congratulations, you have successfully installed the script. You can login with the Admin Email Address and Password you created in the previous step.

Site Configuration

Once logged in as Super admin, you can change "Basic Settings" for the platform.

3.1: User Management

Users cannot register to the platform from the frontend. They must be invited by a site administrator. In the left sidebar:

  • Click on "Team members" to access the list of users in the platform.
  • To invite a new user, click on "Invite user". They will receive an email with an invitation token linked to their email address. They can follow the link in the email to join the platform.
  • The administrator can also "Disable" user accounts, which will prevent the users from logging in again, as well as make other users administrators.

3.2: Logo and Favicon

The logo and favicons are located in the public folder. Simply replace the current ones with yours and that's it.

3.3: Other site settings

In the admin panel, you will also see other settings, including your email settings and site name. Simply update the fields in the form and you're good to go.

Code Design Pattern

We have crafte the source code in a clean way. It uses:

  • Laravel 10
  • Vue.js 3 with Composition API
  • Inertia.js 1.0 (latest version)

The application uses Actions and Data Transfer Objects to keep the code clean and readable. Besides the regular directories that come with Laravel, the following new directories are created: 

            
              |- app
              |---- Actions - contains action classes. Each action is designed to do one thing.
              |---- Data - contains Data Transfer Objects. Used for form validation and formatting request data to PHP objects
              |---- Enums - contains PHP Enums for various data points. Very valuable for readability
              |---- Filters - contains Eloquent query filters used in Pipelines
              |---- Rules - contains custom rules for form validation
              |---- Support - contains various utility classes and helper functions
              |---- Trait - contains Model traits such as traits handling String Primary Key generation

To compile frontend assets, we used yarn. But feel free to use npm if that works better for you.
